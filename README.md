# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`

#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Типы данных и интерфейсы

- `IProduct` – описание товара (id, description, image, title, category, price)
- `IBuyer` – данные покупателя (payment, email, phone, address), где `payment` может быть `'card' | 'cash' | null` (null означает, что способ не выбран)
- `TPayment` – `'card' | 'cash'`
- `IOrder extends IBuyer` – заказ для отправки (добавляет `items: string[]` и `total: number`)
- `IProductsResponse` – ответ сервера со списком товаров (`items: IProduct[]`, `total: number`)
- `IOrderResponse` – ответ сервера после заказа (`id: string`, `total: number`)
- `BuyerValidationErrors` – тип для ошибок валидации (Partial<Record<keyof IBuyer, string>>)

## Модели данных

### ProductModel
Отвечает за хранение каталога товаров и выбранного товара. Принимает в конструктор `EventEmitter` и генерирует событие `catalog:changed` при вызове `setItems`.

**Поля:**
- `private items: IProduct[]` – массив всех товаров.
- `private selectedProduct: IProduct | null` – выбранный товар (для модального окна).

**Методы:**
- `setItems(products: IProduct[]): void` – заменяет массив товаров новым, эмитит `catalog:changed`.
- `getItems(): IProduct[]` – возвращает массив всех товаров.
- `getProductById(id: string): IProduct | undefined` – ищет товар по id.
- `setSelectedProduct(product: IProduct): void` – сохраняет выбранный товар.
- `getSelectedProduct(): IProduct | null` – возвращает выбранный товар.

### CartModel
Отвечает за хранение товаров, добавленных в корзину. Принимает `EventEmitter`, генерирует `cart:changed` при любом изменении состава корзины.

**Поля:**
- `private items: IProduct[]` – массив товаров в корзине.

**Методы:**
- `getItems(): IProduct[]` – возвращает массив товаров в корзине.
- `addItem(product: IProduct): void` – добавляет товар, если его ещё нет в корзине, эмитит `cart:changed`.
- `removeItem(productId: string): void` – удаляет товар по id, эмитит `cart:changed`.
- `clear(): void` – очищает корзину, эмитит `cart:changed`.
- `getTotal(): number` – возвращает общую стоимость (цена null заменяется 0).
- `getCount(): number` – возвращает количество товаров.
- `hasItem(productId: string): boolean` – проверяет наличие товара по id.

### BuyerModel
Отвечает за хранение данных покупателя и их валидацию. Принимает `EventEmitter`, генерирует `buyer:changed` при вызове `setData` или `clear`.

**Поля:**
- `private data: IBuyer` – объект с полями payment, email, phone, address.

**Методы:**
- `setData(partial: Partial<IBuyer>): void` – обновляет только переданные поля, эмитит `buyer:changed`.
- `getData(): IBuyer` – возвращает копию всех данных.
- `clear(): void` – сбрасывает все поля (payment → null, строковые поля → ''), эмитит `buyer:changed`.
- `validate(): BuyerValidationErrors` – проверяет заполненность полей и возвращает объект с ошибками.

## Слой коммуникации

### LarekApi
Отвечает за взаимодействие с сервером. Использует композицию с интерфейсом `IApi` (инъекция зависимости).

**Конструктор:**
- `constructor(api: IApi)` – принимает экземпляр, реализующий методы `get` и `post`.

**Методы:**
- `getProducts(): Promise<IProductsResponse>` – выполняет GET-запрос на `/product`, возвращает объект с массивом товаров.
- `postOrder(order: IOrder): Promise<IOrderResponse>` – выполняет POST-запрос на `/order`, передаёт данные заказа, возвращает ответ сервера.

## Компоненты представления (View)

Все компоненты находятся в папке `src/components/views/`. Каждый класс принимает в конструктор корневой DOM-элемент и (при необходимости) экземпляр `EventEmitter`.

### Page
Корневая страница приложения. Содержит галерею товаров и счётчик корзины. При клике на иконку корзины эмитит `basket:open`.

**Конструктор:** `constructor(container: HTMLElement, events: EventEmitter)`  
**Сеттеры:**
- `catalog: HTMLElement[]` – заменяет содержимое элемента `.gallery`.
- `cartCount: number` – обновляет значение счётчика на иконке корзины.

### Modal
Универсальное модальное окно. Закрывается по клику на крестик, по клику вне окна.

**Конструктор:** `constructor(container: HTMLElement)`  
**Сеттер:**
- `content: HTMLElement | DocumentFragment` – устанавливает содержимое модального окна.
**Методы:**
- `open(): void` – добавляет класс `modal_active`.
- `close(): void` – убирает класс `modal_active`.

### Card (абстрактный)
Базовый класс для карточек товара. Содержит общие элементы: заголовок, категорию, изображение, цену.

**Конструктор:** `constructor(container: HTMLElement)`  
**Сеттеры:** `title`, `category`, `image`, `price` – заполняют соответствующие DOM-элементы.

Наследуется классами `CardCatalog`, `CardPreview`, `CardBasket`.

### CardCatalog
Карточка товара в галерее. При клике эмитит `card:select` с объектом товара.

**Конструктор:** `constructor(container: HTMLElement, events: EventEmitter)`  
**Сеттер:**
- `data: IProduct` – заполняет карточку данными товара.

### CardPreview
Детальная карточка товара, открываемая в модальном окне. Содержит описание и кнопку «Купить» / «Удалить из корзины». При клике на кнопку эмитит `product:toggle`.

**Конструктор:** `constructor(container: HTMLElement, events: EventEmitter)`  
**Сеттеры:**
- `data: IProduct` – заполняет информацию о товаре.
- `description: string` – устанавливает текст описания.
- `inBasket: boolean` – переключает состояние кнопки (Купить/Удалить) и блокирует, если цена null.

### CardBasket
Карточка товара в корзине. Содержит порядковый номер и кнопку удаления (эмитит `basket:remove`).

**Конструктор:** `constructor(container: HTMLElement, events: EventEmitter, product: IProduct)`  
**Сеттеры:**
- `data: IProduct` – отображает название и цену.
- `index: number` – устанавливает порядковый номер.

### Basket
Компонент корзины. Отображает список товаров, итоговую сумму и кнопку «Оформить» (эмитит `order:start`). Если список пуст, показывает заглушку «Корзина пуста» и деактивирует кнопку.

**Конструктор:** `constructor(container: HTMLElement, events: EventEmitter)`  
**Сеттеры:**
- `items: HTMLElement[]` – заменяет содержимое списка товаров.
- `total: number` – обновляет итоговую сумму.

### Form (абстрактный)
Базовый класс для форм. Управляет кнопкой отправки и блоком ошибок.

**Конструктор:** `constructor(container: HTMLFormElement)`  
**Сеттеры:**
- `valid: boolean` – управляет активностью кнопки.
- `errors: string` – устанавливает текст ошибок.
**Метод:**
- `abstract getInputData(): T` – возвращает объект с данными формы.

### OrderForm (extends Form<OrderData>)
Форма первого шага оформления: выбор оплаты (карта/наличные) и ввод адреса. При изменении полей эмитит `order:change`, при сабмите – `order:submit`.

**Дополнительные сеттеры:** `payment`, `address` – для восстановления состояния из модели.

### ContactsForm (extends Form<ContactsData>)
Форма второго шага: email и телефон. При изменении эмитит `contacts:change`, при сабмите – `contacts:submit`.

**Дополнительные сеттеры:** `email`, `phone`.

### Success
Окно успешного оформления заказа. Отображает итоговую сумму. При клике на кнопку закрытия эмитит `success:close`.

**Конструктор:** `constructor(container: HTMLElement, events: EventEmitter)`  
**Сеттер:**
- `total: number` – выводит сумму списания.

## Презентер и события

Презентер реализован в файле `src/main.ts`. Он создаёт экземпляры моделей и представлений, связывая их через подписки на события `EventEmitter`. Основные действия презентера:

1. Загружает товары с сервера и сохраняет в `ProductModel`.
2. Реагирует на событие `catalog:changed` – отрисовывает карточки в галерее.
3. При `card:select` открывает модальное окно с превью товара.
4. При `product:toggle` добавляет/удаляет товар из корзины и закрывает превью.
5. При `cart:changed` обновляет счётчик корзины и (если открыта) содержимое корзины.
6. При `basket:open` открывает модальное окно с корзиной.
7. При `basket:remove` удаляет товар из корзины.
8. При `order:start` очищает данные покупателя и открывает форму первого шага.
9. При `order:change` сохраняет данные в `BuyerModel`, валидирует их и обновляет состояние формы.
10. При `order:submit` (если нет ошибок) открывает форму второго шага.
11. При `contacts:change` сохраняет email/телефон, валидирует и обновляет состояние формы.
12. При `contacts:submit` (если нет ошибок) формирует заказ, отправляет на сервер, при успехе показывает `Success` и очищает корзину и данные покупателя.
13. При `success:close` закрывает модальное окно.

### Перечень событий

| Событие | Источник | Данные | Действие |
|---------|----------|--------|----------|
| `catalog:changed` | `ProductModel` | `IProduct[]` | Отрисовка галереи |
| `card:select` | `CardCatalog` | `IProduct` | Открытие превью |
| `product:toggle` | `CardPreview` | `IProduct` | Добавление/удаление из корзины |
| `cart:changed` | `CartModel` | `IProduct[]` | Обновление счётчика и корзины |
| `basket:open` | `Page` | – | Открытие корзины |
| `basket:remove` | `CardBasket` | `IProduct` | Удаление товара из корзины |
| `order:start` | `Basket` | – | Открытие формы заказа (шаг 1) |
| `order:change` | `OrderForm` | `{ payment, address }` | Валидация и сохранение |
| `order:submit` | `OrderForm` | – | Переход к шагу 2 |
| `contacts:change` | `ContactsForm` | `{ email, phone }` | Валидация и сохранение |
| `contacts:submit` | `ContactsForm` | – | Отправка заказа |
| `success:close` | `Success` | – | Закрытие модального окна |

## Заключение
Проект полностью реализует паттерн MVP, разделяя логику, данные и представление. Благодаря событийной модели компоненты слабо связаны и легко поддаются модификации.
